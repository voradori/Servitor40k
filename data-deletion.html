Data Deletion Instructions

App name: Threads Servitor
Last updated: 1 June 2026

Threads Servitor is a private automation tool for a single account owner. There is no public user registration. This page explains how to revoke access and delete data associated with the app.

1. Revoke access in Meta / Threads (required)

This removes the app’s ability to call the Threads API on your behalf:

  1. Open Threads or Instagram account settings.
  2. Go to Website permissions or Apps and websites.
  3. Find Threads Servitor (or your Meta app display name) and choose Remove / revoke access.

After revocation, stored access tokens stop working for API calls.

2. Delete data on the operator’s systems

The bot may store copies of API-related data locally (not on this GitHub Pages site), for example:

  • OAuth tokens in .env
  • SQLite queue, drafts, or logs on the operator’s computer or server
  • Optional review messages in a private Discord channel

To request deletion of such local data, contact the operator:

  • GitHub Issues — Servitor40k
  • Or the contact email in the repository README

The operator will: invalidate tokens, delete or anonymize local databases and logs related to your Threads account, and confirm when done.

3. Meta data deletion callback

If Meta sends a signed data-deletion request to a configured callback URL on the developer app, the operator will process it within a reasonable time using the same steps above. This static page is for user-facing instructions; it is not an API endpoint.

4. Timeline

Revocation in Threads/Instagram is immediate for new API access. Local deletion is typically completed within 30 days of a verified request unless a shorter period is agreed.
